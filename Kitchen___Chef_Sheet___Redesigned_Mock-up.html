Design Notes — Chef Feedback (27 March 2026) & Changes Made

The following changes were made to the Chef Sheet design in response to feedback from the exec chef, received via Richard Bray on 27 March 2026. Each point below lists what the chef flagged and how it was addressed in this mock-up.

Chef Feedback How Addressed
Page 1 — Guest numbers: "Can they auto fill?" Day, Evening, and Children counts now pull automatically from the Guests tab. No manual entry required.
Page 1 — "Waste of Space" (large blank area after date header) Fixed by the new layout. Each item block sits directly beneath the date header with no gap. The blank page is eliminated.
Page 2 — "Dish name in full" The abbreviated product description beneath the dish name has been removed. The full dish name is the only identifier on each product block.
Page 2 — Dessert marked "Last", Starter marked "First" Course order now runs consistently: Starter (or Grazing Board), Main, Sides, Dessert. Dessert no longer appears first.
Page 2 — "Allergen Info Here" Allergen information is now surfaced at dish level beneath each product block, cross-referencing which named guests eating that specific dish have requirements. Previously only shown in the summary box at the top.
Page 2 — "Menu needs to be set in order" Same as the course ordering fix. Each product block presents courses in logical day order.
Page 2 — "Don't need" (product sub-description beneath dish name) Version description text (e.g. "The Roast — version 2, Includes 1 Starter, 1 Main, 2 Desserts") removed throughout the sheet.
Page 3 — "Dish name in full" (children's menu) Same fix applied to children's menu. Full dish name only, no sub-description.
Page 3 — Pizzas: "Don't need, Make 1 Total" The 70 + 50 (Free) split removed. A single combined quantity of 120 is shown.
Page 3 — "Mornings Breakfast — Number & info" Breakfast now appears as a separate line item derived from accommodation occupancy, showing room count and total covers. Note: will pull automatically from the Accommodation tab once that is built in autumn.
Overall — Black and white printing No colour used anywhere. Time badges, allergen notes, and all UI elements use black, white, and grey only. Prints correctly without colour.
Overall — Right column too long, document too many pages The five-column table layout has been replaced with a heading bar layout. Time, Itinerary Item, Product, and Qty now form a compact single-row header. All course detail runs full-width beneath it. Vastly reduces page count.
